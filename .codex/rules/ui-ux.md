# UI/UX Rules

- Use shadcn-style primitives from `src/components/ui` before custom markup.
- Use semantic Tailwind tokens such as `bg-background`, `text-muted-foreground`, and `border-border`.
- Use `cn()` for class merging and Lucide named imports for icons.
- Keep UI minimal, accessible, responsive, and setup-focused.
- Prefer clear form states, empty states, pending states, and validation messages over decorative complexity.
- For execution details, use the `ui-ux` skill to route to `create-page`, `create-ui-component`, `white-label-app`, or `create-hook`.
