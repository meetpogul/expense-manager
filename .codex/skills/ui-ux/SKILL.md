# UI/UX

1. Read `.codex/rules/ui-ux.md`.
2. Use `create-page` for routes/screens.
3. Use `create-ui-component` for forms, lists, cards, states, and shadcn-style components.
4. Use `white-label-app` for branding, theme, manifest, metadata, icons, or PWA identity.
5. Use `create-hook` when UI state or URL parsing should be extracted.
6. Review final UI for accessibility, responsive layout, semantic tokens, and test coverage.
